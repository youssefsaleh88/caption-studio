# Information Architecture: Caption Studio — واجهة المنتج

**المجلد:** `.design/caption-studio-ui/`  
يستند إلى: `DESIGN_BRIEF.md` والمسارات والمكوّنات الحالية في `frontend/src`.

---

## Site Map

التاريخ مع **React Router**؛ لا توجد مسارات متداخلة متعددة المراحل؛ الجلسة تُحمَّل بحالة التوجيه وليس باراميترات URL.

```
/                          Home.jsx          الصفحة الرئيسية، رفع + لغة فيديو
/editor                    Editor.jsx       المحرّر (يتطلّب state من انتقال داخلي)
```

- **`/`** — نقطة الدخول الوحيدة المستدامة؛ منها يحدث كل التدفّق الناجح.
- **`/editor`** — عنوان صالح بعد `navigate("/editor", { state: { videoUrl, words } })`؛ **فتح مباشر** أو تحديث من دون الحالة الحالية → إعادة توجيه إلى `/` مع `replace` (تفادي جلسات مكسورة).

لا توجد اليوم مسارات تحت مسارات؛ لا حساب إعدادات، لا مساعدة، لا `/settings`.

## Navigation Model

### Primary navigation — يُعتبر «شريط المعنى الرئيسي» وليس قائمة أفقية ثابتة

| السياق | العنصر | ملاحظة |
|--------|--------|--------|
| **Home** | `BrandHeader` (شعار، تطبيق، لغة) + إمكان رجوع غير موجودة هنا بالمنطق | تركيز واحد على منطقة الرفع |
| **Editor — موبايل** | ضمن المحتوى: **معاينة** ثم **زر «المظهر والخط…»** ثم **تبويبان: تحرير / تصدير** | لا قائمة همبرغر؛ عمق تنقل وظيفي 1 مستوى + ورقة مشروطة |
| **Editor — desktop (`lg+`)** | ثلاثة أعمدة ثابتة المسار بصريًا (`dir=ltr`): **فيديو \| إعدادات+تصدير \| جمل** | تنقل عمودي داخل عمود الإعدادات عبر تبويبات لوحة النمط |

### Secondary navigation — تبويبات وتفاصيل

- **تابات المحرّر (موبايل فقط):** تحرير، تصدير.
- **تابات لوحة النمط (`StylePanel`):** خط، خلفية، مؤثرات، موضع، ترجمات (المصطلح الحالي في i18n).

### Utility navigation — أدوات عابرة الهيكل الرئيسي

- **LanguageSwitcher:** في الهيدر — يعرِّف RTL/LTR لكل الصفحة.
- **Instagram + حقوق:** في الفوتر — روابط خارجية محدّدة؛ لا جزء من تدرّج المهمّات.
- **رجوع في المحرّر:** إلى `/`؛ يكمُّل جلسة المحرّر من ناحية التدفّق (المستخدم يعيد الرفع لو احتاج نفس الفيديو لاحقًا — خارج نطاق v1 للحفظ).

### Mobile navigation إجمالًا

ليست قائمة أسفل شاشة عالمية؛ **تكديس عمودي + تبويبان + ورقة تنبثق** لتوسيع المساحة للإعدادات بما يتماشى مع Brief (مبدأ المحتوى أولًا).

## Content Hierarchy — حسب الصفحة

### Home `/`

1. **رأس العلامة + تبديل اللغة** — سياق وثقة وفوري.
2. **العنوان الرئيسي + دعم Gemini** — لماذا الأداة.
3. **منطقة الرفع مع اختيار لغة الفيديو** — أعلى احتمال تفاعل؛ يحدد مسار الدقة قبل التحليل.
4. **نص تمهيدي ثانوي** — أقل الأولوية.
5. **الفوتر** — تأكيد حقوق ومِلّاك.

### Editor `/editor`

**موبايل (قراءة من أعلى لأسفل):**

1. **المعاينة** — مرجع بصري ووقتي للكلام؛ قصّ مقصود لمنع تجاوز المنطقة.
2. **دعوة لفتح الإعدادات** — وصول واضح دون الزحمة في تبويب ثالث.
3. **تبويبا تحرير / تصدير** — يحددان الشريط الآني للمحتوى.
4. ضمن **تحرير:** قائمة بطاقات جمل؛ توسُّع لكلمات وتوقيت؛ زر تشغيل من هنا مرسوم لكل بطاقة.
5. ضمن **تصدير:** أفعال تنزيل SRT وحَرْق؛ مساحة رسالة الخطأ وإعادة المحاولة (حسب Brief).

**ديسكتوب (`lg+`):**

1. **عمود الفيديو** (الأعرض) — المرجع الأهم بصريًا.
2. **عمود الإعدادات وتصدير** — يغيّر العرض المركّب قبل التصدير النهائي.
3. **عمود قائمة الجمل** — حيث يُقضى معظم الزمن المعرفي لتعديل التوقيت وفق قرار Brief.

## User Flows

### Flow أ — مسار أساس سعيد (رفع → تحرير → تصدير)

1. يصل إلى `/`.
2. يختار لغة التفريغ للفيديو (اختياري) ويجري الرفع؛ حالة انتظار تفريغ.
3. عند النجاح: انتقال إلى `/editor` مع `{ videoUrl, words }`.
4. يراجع الحلقات في تبويب **تحرير**؛ يفتح بطاقات ويُصحّح نصًا/تواقيت؛ يطلق «تشغيل من هنا» عند الحاجة.
5. يعدِّل الإعدادات: موبايل عبر ورقة الإعدادات، ديسكتوب في عمود مستمر.
6. ينتقل إلى **تصدير**؛ ينزل SRT أو يطلق تصدير فيديو؛ نجاح = ملف أو تحميل متصفّح حسب المنفّذ.

### Flow ب — وصول مخالف لمحرّر بدون حالة صالحة

1. مستخدم يفتح `/editor` مباشرة أو يصفِّح بعد فقدان `state`.
2. `Editor` يتحقق؛ يوجِّه إلى `/` بـ `replace`.
3. نتيجة: العودة لنقطة الرفع؛ لا رسالة خطأ مستقلة حاليًا (تحسّين محتمل لاحقًا: toast «ابدأ من رفع فيديو»).

### Flow ج — خطأ شبكة أو تصدير

1. المستخدم على تبويب **تصدير** ويجري تنزيل SRT أو فيديو؛ يحدث خطأ.
2. تُعرض **رسالة واضحة داخل المنطقة** (حاليًا عبر مساحة خطأ في `ExportBar`/الطلب؛ يتوافق مع Brief لو أُضيف **إعادة محاولة صريحة** لاحقًا دون مغادرة التبويب).
3. لا رابط خارجي للدعم في v1 افتراضيًا.

### Flow د — الرجوع من المحرّر

1. المستخدم يضغط رجوع في الهيدر.
2. انتقال إلى `/`؛ **لا يُعبَّأ المحتوى تلقائيًا بالفيديو السابق** (لا مشاريع محفوظة في v1).

### Flow هـ — تبديل اللغة

1. من أي صفحة: تبديل AR/EN.
2. حدود التأثير: نص الواجهة واتجاه المستند؛ **لا توجد اليوم باراميترات URL له** — الحالة موجودة محليًا ضمن إعداد الترجمة (i18n).

## Naming Conventions

مفاهيم منتظمة بين العربية والإنجليزية وفق ملفات `i18n`؛ التزم بنفس المصطلح في النسختين.

| المفهوم | عربي (تقريبي) | English | ملاحظة |
|---------|----------------|---------|---------|
| جلسة رفع وحدة | لا «مشروع» | لا Project | خارج النطاق v1 |
| قائمة وقتية للكلمات | ترجمات / جمل | Captions | وضع «جمل» مقابل «مجموعات كلمات» موجود في الواجهة |
| إعداد الشكل والخط واللون | المظهر… / النمط | Style / Look | زر الوصول للورقة مقابل تبويبات اللوحة |
| إخراج نص متحرّك | تصدير / تحميل SRT | Export / Download SRT | أفعال واضحة |
| حقول زمن مقطع | بداية/نهاية جملة | Sentence start/end | مطابقة لـ `caption.*` |

## URL Strategy

| القاعدة | التطبيق الحالي |
|---------|----------------|
| أساس واحد SPA | كل المسارات من جذر التطبيق |
| مسارات اسمية مستقلة | `/` ، `/editor` فقط |
| لا معرفات بالمسار للفيديو | الحالة تأتي من `history.state` لا من `/editor/:id` |
| لا query strings لتصفية قائمة ترجمات | محتمل مستقبلًا للمشاركة العميقّة خارج نطاق v1 |

**ملاحظة معمارية:** استمرار اعتماد `state` فقط يعني لا روابط تشاريك مباشرة للمحرّر؛ مقبول لـ Brief «جلسة واحدة».

## ملخص قرارات لم تُطرح منفردة هذه الجولة

مستخلص ضمنياً من الـ Brief + الكود؛ يُمكن مراجعته لاحقًا:

1. **عمق التنقل المقبول:** مستويان كحدّ أقصى شائع (صفحة ثم تبويب أو ورقة)؛ عمود الثلاثة على الديسكتوب ليس عمق تنقلًا بل تخطيطًا متزامنًا.
2. **صفحة حيث يُفترض أن يقضي المستخدم ~80٪ من الزمن المعرفي:** **المحرّر**؛ على الموبايل معظمها داخل **تبويب التحرير**.
3. **أنواع مستخدمين مختلفة لنقاط دخول:** لا — شخصية واحدة (صانع محتوى موبايل أولًا) وفق المرحلة 1.
